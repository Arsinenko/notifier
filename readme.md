

# Folder Watcher & Notifier

Сервис для мониторинга изменений в файловой системе с последующим уведомлением пользователей. Система позволяет гибко настраивать права доступа: уведомления получают только те пользователи, которым администратор выдал соответствующие разрешения на конкретные типы событий.

## 🚀 Основные возможности

*   **Мониторинг папок:** Отслеживание создания, изменения, удаления и переименования файлов в реальном времени.
*   **Уведомления:** Поддержка отправки алертов через Telegram и Email.
*   **Управление пользователями:** Встроенная Web-панель для администрирования прав доступа.
*   **Гибкая настройка:** Система разрешений (Permissions) для каждого отдельного пользователя.

## 🛠 Установка и запуск

1. **Клонируйте репозиторий:**
   ```bash
   git clone <repository_url>
   cd notifier
   ```

2. **Настройте конфигурацию:**
   Создайте файл `config/config.yaml` (см. раздел "Конфигурация").

3. **Запустите приложение:**
   ```bash
   go run main.go
   ```

4. **Доступ к панели управления:**
   Откройте браузер по адресу: `http://localhost:8080`

## ⚙️ Конфигурация

Проект использует файл `config.yaml` для настройки окружения.

### Требования к конфигу:

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `token` | string | API токен вашего Telegram бота (получается у @BotFather). |
| `mail.server` | string | SMTP сервер для отправки почты (например, `smtp.gmail.com`). |
| `mail.port` | int | Порт SMTP сервера (обычно 587 или 465). |
| `mail.email` | string | Email адрес отправителя. |
| `mail.password` | string | Пароль или App Password от почтового ящика. |
| `target_folder` | string | Путь к директории, за которой нужно следить. |
| `users_filepath` | string | Путь к файлу `users.json`, где хранятся данные пользователей. |

**Пример `config.yaml`:**
```yaml
token: "123456789:ABCDefGhIJKlmNoPQRstUv"
target_folder: "./monitored_folder"
users_filepath: "./users.json"
mail:
  server: "smtp.gmail.com"
  port: 587
  email: "your-app@gmail.com"
  password: "your-password"
```

## 🔐 Система разрешений (Permissions)

Уведомления рассылаются на основе разрешений, назначенных пользователю. В системе предусмотрены следующие типы прав:

*   `CREATE` — Уведомлять при создании новых файлов или директорий.
*   `WRITE` — Уведомлять при изменении содержимого существующих файлов.
*   `REMOVE` — Уведомлять при удалении файлов или папок.
*   `RENAME` — Уведомлять при переименовании объектов.

## 🖥 Web-интерфейс

Администрирование осуществляется через локальный веб-интерфейс:
*   **Адрес:** `http://localhost:8080`
*   **Функционал:** Просмотр списка пользователей, добавление/удаление пользователей, управление правами на уведомления (CREATE, WRITE, REMOVE, RENAME).
